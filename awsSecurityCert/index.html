Domain 1: Incident Response - 12%

Given an AWS abuse notice, evaluate the suspected compromised instance or exposed access keys.

Evaluate the configuration of automated alerting, and execute possible remediation of security-related incidents and emerging issues.

  • IAM
  • AWS Config
  • AWS GuardDuty
  • AWS Cloudtrail
  • Cloudwatch

Domain 2: Logging and Monitoring - 20%

Design and implement security monitoring and alerting.

Troubleshoot logging solutions

  • IAM
  • Cloudwatch Events
  • Cloudwatch Insights
  • AWS Config
  • AWS GuardDuty
  • AWS Cloudtrail
  • Athena
  • VPC Flow logs
  • Cloudfront and WAF logs

Domain 3: Infrastructure Security - 26%

Design edge security on AWS.

Design and implement a secure network infrastructure.

Troubleshoot a secure network infrastructure.

  • ACL
  • Security Groups
  • IAM permissions
  • API Gateway
  • VPN
  • VPC Endpoints

Domain 4: Identity and Access Management - 20%

Design and implement a scalable authorization and authentication system to access AWS resources.

Troubleshoot an authorization and authentication system to access AWS resources.

  • IAM permissions (policies, evaluation, conditions)
  • S3 permissions
  • Federation
  • Cognito
  • API Gateway
  • AWS Organizations (SCP)

Domain 5: Data Protection - 22%

Design and implement a scalable authorization and authentication system to access AWS resources.

Troubleshoot key management.

Design and implement a data encryption solution for data at rest and data in transit.

  • KMS!!
  • Cloud HSM
  • Federation
  • S3
  • Data at rest and in transit

Not covered

  • Security attacks
  • DAST and SAST
  • Common Vulnerabilities and Exposures (CVE)
  • OAUTH authentication vulnerabilities
